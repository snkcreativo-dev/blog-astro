Este blog existe para documentar decisiones tecnicas con claridad.

## La idea principal

No se trata solo de publicar articulos. Tambien se trata de que el propio proyecto sea legible:

- estructura predecible
- decisiones documentadas
- despliegue reproducible
- flujo editorial entendible

## Lo que intenta evitar esta base

- dependencias innecesarias
- acoplamiento entre contenido y interfaz
- configuraciones opacas
- soluciones que funcionan pero no se entienden

## Perfil ideal de uso

Esta base encaja bien si quieres:

- aprender Astro en un proyecto real
- tener un blog editable sin WordPress
- desplegar gratis en GitHub Pages
- mantener control total sobre el contenido en Git
